## v3.1.2 — исторический визуальный архив

Для карточек без вручную закреплённого изображения приложение использует API `PageImages` русской и английской Wikipedia. После выбора статьи оно запрашивает `ImageInfo + extmetadata` на Wikimedia Commons и сохраняет:

- уменьшенный URL изображения;
- название файла;
- автора или кредит;
- краткое название лицензии;
- ссылку на страницу файла;
- дату разрешения изображения.

Эти данные хранятся только до конца текущей сессии в `sessionStorage` и показываются в карточке. Условия повторного использования определяются лицензией конкретного файла на Wikimedia Commons. Если изображение или API недоступны, используется уникальная локальная SVG-обложка карточки.

`data/image_queries.json` содержит 1139 поисковых профилей: 42 карточки используют закреплённые изображения, 1097 — динамический семантический подбор. Один резервный файл не назначался более чем девяти новым карточкам, кроме точных совпадений темы.


## v3.1.4 — семантическая проверка

Перед сохранением динамического изображения приложение проверяет описание, вводный текст и категории статьи. Совпадение должно относиться к историческому региону и типу карточки. Биологические, медийные, коммерческие и другие одноимённые темы отклоняются.

Для 80 неоднозначных карточек заданы отдельные статьи. В частности, «Тигр» ведёт к реке Tigris, а римская «Коза» — к колонии Cosa. При отсутствии надёжного совпадения используется локальная учебная обложка. Контекстный файл назначается максимум восьми карточкам.
