# Codex of History — MVP Architecture

## Принцип

Проект строится как исторический content engine.

Не так:

```text
страница кампании → текст → вопросы → всё жёстко внутри
```

А так:

```text
cards → relations → campaigns → nodes → missions → quizzes → progress
```

История хранится в карточках.  
Кампании только строят маршрут.  
Миссии только задают условия.  
Сюжет только объясняет и ведёт.  
Квизы проверяют знание.  
Прогресс пользователя хранится отдельно.

---

## Главные сущности

### Card
Атом системы.

Хранит:
- название;
- тип;
- дату;
- эпоху;
- регион;
- краткую суть;
- важность;
- факты;
- статы;
- редкость;
- источники;
- локацию.

### Relation
Связь между карточками.

Примеры:
- CAUSE
- CONSEQUENCE
- PART_OF
- COMMANDER_IN
- INFLUENCED
- PRECEDES

### Campaign
Маршрут по карточкам.

Кампания не копирует контент.  
Она ссылается на карточки.

### Campaign Node
Глава/узел кампании.

Типы:
- INTRO
- LESSON
- BATTLE_STUDY
- STORY_SCENE
- BOSS_QUIZ

### Mission
Декларативная задача.

Пример:
- прочитать 3 карточки;
- пройти квиз на 70%;
- открыть связь;
- найти событие на карте.

### Quiz
Набор вопросов.

Один квиз можно использовать:
- в карточке;
- в уроке;
- в кампании;
- в повторении.

### Source / Citation
Фактчек.

В идеале каждый важный факт должен иметь источник.

---

## Почему это масштабируется

Можно добавить новую кампанию без кода.

Нужно добавить:

```text
campaign
campaign_nodes
node_cards
missions
quizzes
story_episodes
```

Карточки уже лежат в общей базе.

---

## Минимальный backend

Для Supabase/PostgreSQL:

1. выполнить `schema.sql`;
2. добавить справочники eras/regions;
3. импортировать cards;
4. импортировать relations;
5. импортировать campaigns;
6. импортировать quizzes;
7. подключить мобильный клиент.

---

## Масштаб

Система спокойно растёт до:

- 1 000 карточек;
- 10 000 карточек;
- 50 000 карточек.

Критичные места:

- поиск;
- фильтры;
- граф связей;
- картинки;
- пользовательский прогресс.

Решение:

- PostgreSQL indexes;
- JSONB только для гибких полей;
- отдельные relations;
- lazy loading графа;
- pagination;
- кэш текущей кампании на телефоне.
