# Codex of History — Daily Learning Architecture v1.2

## Данные

Настройки находятся в `data/core/daily-learning.json`. Движок загружает их через общий content manifest.

## Состояние пользователя

В `localStorage` добавлены:

- `reviewSchedule` — интервал и дата следующего повторения каждой карточки;
- `dailyHistory` — результаты завершённых дневных сессий;
- `learningDays` — уникальные учебные дни;
- `dailyStats` — общая статистика ответов;
- `dailySession` — незавершённая или завершённая сессия текущего дня.

## Алгоритм

1. Новая прочитанная карточка получает первое повторение через день.
2. Старые сохранения без расписания получают стартовую очередь на текущий день.
3. Сессия выбирает просроченные карточки, затем добирает самые слабые изученные.
4. Правильный ответ двигает карточку на следующий интервал.
5. Ошибка ставит повторение на следующий день.
6. После сессии открывается бесплатный архивный пак.

## Масштабирование

Расписание хранится по `card_id`, поэтому система работает независимо от кампании и количества карточек. Новые эпохи автоматически участвуют в повторениях после чтения их карточек.
