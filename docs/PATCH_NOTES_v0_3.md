## Quiz Logic

Раньше квиз попадал в `quizDone` после завершения последнего вопроса, даже если ответы были неправильные.

Теперь:

- квиз имеет порог зачёта `70%`;
- результат хранится в `quizResults`;
- прогресс кампании считает только зачтённые квизы;
- следующая глава открывается только после зачёта;
- в кампании показывается лучший процент по главе;
- после квиза показывается результат: `score / total`, процент, статус.

## LocalStorage

Старые записи `quizDone` остаются для совместимости, но новая логика кампании опирается на `quizResults[quizId].passed`.
Если в старом браузере были «закрытые» квизы без результата, они могут снова отображаться как незачтённые. Это нормально: старый MVP не сохранял процент.
