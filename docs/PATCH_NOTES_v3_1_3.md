# Patch v3.1.3 — семантическая проверка изображений

## Причина

Визуальный модуль v3.1.2 принимал первое совпадение статьи Wikipedia только по названию. Поэтому карточка реки «Тигр» могла получить фотографию животного, а римская колония «Коза» — домашнюю козу.

## Исправления

- удалены случайные кандидаты из общих пулов кампаний;
- для 899 карточек пересобраны поисковые профили;
- добавлены 80 ручных развязок неоднозначных названий;
- 307 коротких, общих и рискованных тем переведены в строгий режим;
- перед назначением изображения проверяются:
  - вводный текст статьи;
  - описание Wikidata/PageTerms;
  - категории статьи;
  - тип карточки;
  - исторический регион;
  - запрещённые современные и биологические значения;
- неподходящее совпадение отклоняется;
- если надёжного изображения нет, остаётся локальная SVG-обложка;
- старый кэш v3.1.2 автоматически удаляется;
- Service Worker и runtime обновлены до v3.1.3.

## Примеры исправленных коллизий

- Тигр → `Тигр (река)` / `Tigris`;
- Коза → `Cosa`, римская колония;
- Ка → `Ka (pharaoh)`;
- Мари → древний город;
- Киш → шумерский город;
- Мемфис → город в Египте;
- Инд и Рави → реки;
- У-ван → правитель Чжоу;
- Регия, Фасты, Курии и Целий → римский контекст.

## Проверки

- content validator;
- статическая проверка 899 профилей;
- отсутствие случайных fallback-кандидатов;
- runtime-тест с ложными страницами тигра и козы;
- подтверждение выбора страниц реки Тигр и римской Козы/Cosa;
- проверка локальных fallback-файлов;
- проверка единой версии и Service Worker.
