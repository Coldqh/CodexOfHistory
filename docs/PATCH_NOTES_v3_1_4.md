# Patch v3.1.4 — Mobile PWA stability

## Причина сбоя

Версия 3.1.3 при запуске начинала обход 857 динамических изображений и отдельно прогревала изображения всех 899 карточек. Управляемый Service Worker сохранял изображения и все ответы Wikipedia/Wikimedia API без лимита. В установленном iOS web app это могло переполнять память процесса и Cache Storage.

## Исправление

- ленивый подбор только для карточек в области просмотра;
- автоматические партии по 4 карточки в standalone и 10 в браузере;
- ручные партии по 12/36 карточек;
- максимум 72 визуальные записи в localStorage на standalone;
- ограниченные кэши изображений и тайлов;
- API-ответы не кэшируются Service Worker;
- ранняя очистка старых кэшей;
- PNG PWA-иконки и облегчённый standalone CSS.
